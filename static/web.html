project 00

the a-hills

When my household decided to build a home climbing wall, I instantly knew it was the perfect opportunity to put into practice the web development skills I was learning.

The website features a log-in for those who climb on the wall, and forms to input and access a database of climbing problems. I've also developed an admin panel to update user permissions, and am working on more interactive features, such as leaving reviews for each problem and searching by feature.

home page

The A-hills home page is a clean beauty shot of the climbing wall, with a sticky nav bar allowing access to the important features. The "climb on" button prompts the user to scroll to the main content.

login form

The user sign up and login forms pop-up in the corner of the page; the login also features a fully automated request for password reset.

admin panel

The admin panel is exposed using a button that only members who have admin permissions can access. The simple form allows the admin user to easily delete a user (with a confirmation), or update a user's permissions.

current problems display

Problems listed as current in the database are automatically displayed in the main area of the page upon page load. A future release will allow the user to toggle between card view and tabular view, as well as providing a rating.

contact & top icons

The client requested a scroll-to-top icon, and also wanted to include social media and contact links using branded icons that stayed on the homepage. The Wordpress theme that was chosen didn't allow for the client's request without adding additional costs. My solution was to create a custom widget using the icons the client requested and floating them on a side panel.

downloadable media

A resources page features buttons to download the client's print media, which I also helped create to coordinate with the website.
